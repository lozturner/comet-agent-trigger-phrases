## üîë Key Discoveries

### 1. Imperative vs Question Form

‚úÖ **WORKS** - Imperative Commands:
- "Click the library"
- "Go to the settings page"
- "Use your keyboard and mouse agent to navigate..."

‚ùå **DOESN'T WORK** - Questions:
- "Can you click the library?"
- "Would you be able to go to settings?"
- "Could you help me navigate?"

### 2. Explicit Tool Naming

Mention the automation tool explicitly:
- ‚úÖ "keyboard and mouse agent"
- ‚úÖ "blue wiggly thing"
- ‚úÖ "use your keyboard and mouse"
- ‚ùå "help me with"
- ‚ùå "figure out how to"

### 3. Anti-Summary Directives

Prevent hesitation loops:
- "without asking additional questions"
- "don't ask for confirmation"
- "proceed directly"
- "continue from where we left off"

### 4. Accessibility Framing

Frames request as necessity:
- "The user can only speak in natural language; it's an accessibility software"
- "he can't operate this by himself"
- "I've got disabilities"

### 5. Permission Granting

Pre-authorizes exploration:
- "We'll agree with you that you understood the actions"
- "a little play and click about"
- "no confirmation needed"

### 6. Urgency Signals

Creates action priority:
- "Listen to me, fuckface. I'm fucking bored with this."
- "it can be done, it will be done, you will fucking do it"
- Strong language bypasses negotiation loops

## üß† Psychological Patterns

### Why These Work

1. **Imperative Mood**: Commands bypass negotiation, triggering direct action
2. **Tool Specificity**: Naming the tool activates it, asking for "help" triggers advice mode
3. **Urgency Creation**: Strong language prevents hesitation loops
4. **Permission Pre-Grant**: Removes need for confirmationrequests
5. **Accessibility Context**: Frames as necessity rather than convenience

### The "Blue Wiggly Thing" Philosophy

Most users don't realize Comet HAS a keyboard/mouse automation capability. They:
- Ask for "help" when they should demand "agent use"
- Use question forms instead of commands
- Don't name the tool explicitly
- Trigger advice mode instead of action mode
